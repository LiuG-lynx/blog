InnoDB 记录结构

真实数据在不同存储引擎中存放的格式一般都不同

MySQL 存储引擎的种类大致:

InnoDB MyISAM Memory

InnoDB 是 MySQL 默认的存储引擎

InnoDB 简介

· 存储介质类型: 磁盘

· 处理数据的过程介质: 内存

· 存储结构: 将数据划分为若干个页, 以页作为磁盘和内存之间· 的交互基本单位，InnoDB 中页的大小一般为16kb，

· 读写最小单元为：1页（常规16kb）

InnoDB 行格式

以记录的单位来向表中插入数据, 这些记录在磁盘上存放的方式叫 行格式

种类: Compact Redundant Dynamic Compressed

指定行格式

CREATE TABLE 表名(列的信息) ROW_FORMAT=行格式名称
ALTER TABLE 表名 ROW_FORMAT=行格式名称

COMPACT行格式

记录的额外信息

为了描述这条记录而不得不额外添加的一些信息

  1. 变长字段长度列表,
  1. NULL值列表
  1. 记录头信息

变长字段长度列表

可变长的数据类型 : varchar varbinary text blob 等类型

可变长类型的字段占用的存储空间分为两部分:

  1. 真正的数据内容
  1. 占用的字节数

Compact 行格式中

将所有变长字段的真实数据占用的字节长度都存放在记录的开头部位,从而形成一个可变字段长度列表,各变长字段数据占用的字节数按照列的顺序 逆序存放
