四个 name 分别代表什么

Deployment

允许我们进行扩容 滚动更新或降级操作

执行kubectl run redis --image='redis:alpine' 创建一个 名为 redis 的 Deployment 并为其指定 redis:alpine 镜像

deployment 将Pod 托管给 replicaSet

ReplicaSet 简写 rs

低级别结构 允许扩容

rs 有 deployment 创建 不推荐主动自行创建

Service

稳定的入口访问我们的应用服务或者一组Pod 通过 Service 可以很方便的实现 服务发现和负载均衡

root@ks-allinone /h/z/docker_space# kubectl get service -o wide
NAME         TYPE        CLUSTER-IP   EXTERNAL-IP   PORT(S)   AGE   SELECTOR
kubernetes   ClusterIP   10.96.0.1    <none>        443/TCP   31m   <none>

Service 的4种类型

  1. ClusterIP 是 k8s 的service 的默认类型 将 service 暴露于一个仅集群内可访问的 虚拟IP上
  1. NodePort 是通过在进群内所有 Node 上都绑定固定端口的方式将服务爆楼出来 可通过<NodeIP>:<NodePort> 访问服务了
  1. LoadBalancer 是通过 Cloud Provider 创建一个外部的负载均衡器，将服务暴露出来，并且会自动创建外部负载均衡器路由请求所需的 Nodeport 或 ClusterIP 。
  1. ExternalName 是通过将服务由 DNS CNAME 的方式转发到指定的域名上将服务暴露出来，这需要 kube-dns 1.7 或更高版本支持。

实践 将刚部署的 redis 暴露服务

root@ks-allinone /h/z/docker_space# 
kubectl expose deploy/redis --port=6379 --protocol=TCP --target-port=6379 --name=redis-server
service/redis-server exposed
root@ks-allinone /h/z/docker_space# kubectl get svc -o wide
NAME           TYPE        CLUSTER-IP      EXTERNAL-IP   PORT(S)    AGE   SELECTOR
kubernetes     ClusterIP   10.96.0.1       <none>        443/TCP    38m   <none>
redis-server   ClusterIP   10.96.192.166   <none>        6379/TCP   14s   run=redis

由于 redis 默认类型 ClusterIP 所有不能直接通过外部进行访问 使用 port-forward 方式让他可以在集群外部访问

一次性访问

# 由于本地6379 端口冲突我更换了一下外部访问的端口
# 此方法仅是开辟一个在 终端上的 服务 开启另外一个终端即可使用 redis访问
kubectl port-forward svc/redis-server 6379:6379

切换为NodePort 对外持久访问

root@ks-allinone /h/z/docker_space# 
kubectl expose deploy/redis --port=6379 --protocol=TCP --target-port=6379 --name=redis-server-nodeport --type=NodePort
service/redis-server-nodeport exposed
root@ks-allinone /h/z/docker_space# kubectl get service/redis-server-nodeport -o wide 
NAME                    TYPE       CLUSTER-IP     EXTERNAL-IP   PORT(S)          AGE   SELECTOR
redis-server-nodeport   NodePort   10.96.70.203   <none>        6379:30774/TCP   10s   run=redis

我们可以通过任意 Node 上的 30774 端口连接

需要注意的是 端口 可以通过 kube-apiserver 的 service-node-port-range 进行配置 范围 默认 30000-32767
